TaskFlow is built on a decoupled architecture. The frontend is a modern Angular Single Page Application (SPA) that communicates with the backend exclusively via RESTful JSON endpoints. Authentication is completely stateless, passing an encrypted JSON Web Token (JWT) in every HTTP header.

Soft Delete Mechanism

Important records like projects and tasks use soft deletes (is_deleted) rather than being permanently removed. This prevents data fragmentation and preserves the integrity of the audit trails.

Dynamic Kanban Columns

Unlike traditional systems with hardcoded statuses, each project has its own rows in project_statuses. Tasks join with these custom statuses via the unique slug identifier.
